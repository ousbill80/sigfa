**Fichiers pressentis** (agent-api) : routes `POST /reports/export` (202 + jobId) + `GET /reports/export/:jobId` conformes CONTRACT-006 · `apps/api/src/jobs/export-build.job.ts` (génère le fichier, écrit `file_url` signé + `expires_at`) · `apps/api/src/reporting/benchmark.ts` (classement) · route `GET /reports/benchmark`.

## Benchmarking — définition EXACTE

- Classement des agences d'un tenant sur un **KPI de tri** (défaut : Taux SLA, configurable ∈ {TMA, taux SLA, NPS, abandon}) sur une période donnée, KPIs calculés via REP-001.
- **Statut de couleur** (seuils DB-003 / config tenant, documentés) : `vert` = KPI ≥ objectif ; `orange` = entre objectif et seuil d'alerte ; `rouge` = < seuil d'alerte. Sens du « meilleur » dépend du KPI (SLA/NPS : plus haut = mieux ; TMA/abandon : plus bas = mieux) — normalisation documentée.
- Agences sans donnée sur la période → statut `n/a` (jamais classées comme rouge par défaut).

## Exigences (EARS)

- **UBIQUITAIRE** — Tous les chiffres d'export et de benchmarking sont dérivés de **REP-001** (mêmes formules) — aucune formule réimplémentée.
- **ÉVÉNEMENT** — QUAND `POST /reports/export?format=pdf|xlsx|json&scope=&period=` est appelé (rôle DIRECTOR+/AUDITOR uniquement — AGENT interdit), le système crée une entrée `export_jobs` (statut `PENDING`), enfile un job BullMQ, et retourne **202 + `jobId`** (génération asynchrone contractualisée CONTRACT-006).
- **ÉVÉNEMENT** — QUAND le job d'export s'exécute, il génère le fichier au format demandé, écrit une **URL de téléchargement signée** + `expires_at` dans `export_jobs`, et passe le statut à `READY` (`PENDING→PROCESSING→READY|FAILED`, enum LA LOI DB-006).
- **ÉVÉNEMENT** — QUAND `GET /reports/export/:jobId` est appelé par le demandeur, le système retourne le statut et, si `READY`, l'URL signée et son expiration.
- **ÉVÉNEMENT** — QUAND `GET /reports/benchmark?scope=network&period=&sortKpi=` est appelé, le système retourne le classement des agences avec statut vert/orange/rouge/n-a par agence, en **agrégats anonymisés** (`AnonymizedNetworkAggregate`).
- **ÉTAT** — TANT Que le scope d'un export est `network`, le contenu exporté ne comporte AUCUNE donnée personnelle (anonymisation UEMOA documentée), quel que soit le format.
- **INDÉSIRABLE (ownership)** — SI un utilisateur interroge/télécharge un `jobId` qui ne lui appartient pas (ou hors tenant), ALORS 404 opaque (pas d'oracle d'existence cross-tenant).
- **INDÉSIRABLE (expiration)** — SI l'URL signée est expirée (`NOW() > expires_at`), ALORS le téléchargement est refusé et le statut reflète l'expiration (regénération = nouvel export).
- **INDÉSIRABLE (échec génération)** — SI la génération échoue, ALORS `export_jobs.status = FAILED` avec cause journalisée ; retry/dead-letter selon NOTIF-001 ; jamais un fichier corrompu servi.
- **INDÉSIRABLE (JSON = contrat)** — L'export JSON respecte le schéma d'export de CONTRACT-006 (mêmes clés/unités que `/reports/kpis`) — contractuel, pas ad hoc.
