## TV-001 : Layout écran d'appel TV — héros 180px + 3 précédents + file

**Agent** : agent-web · **Module** : F4 · **Dépend de** : CONTRACT-009, CONTRACT-012 · **Statut** : TODO

**Révision** : v2 — arbitrage 19

> DESIGN-gate humain obligatoire avant implémentation (PRD §4, règle 5).
> Code contre le mock Prism + événements simulés — sockets réels = RT-001.

---

### Exigences (EARS)

**UBIQUITAIRE** : L'écran doit afficher en permanence sur fond `--surface-screen` (#0A0F1A) :
- Zone héros : dernier ticket appelé en `ticket-called` **180px minimum** (token Design System), guichet en **120px**, typographie "Clash Display" / "General Sans" semibold, couleur `--ink-inverse`.
- Rangée précédents : 3 appels précédents en **64px minimum** (token `--display-tv` ≥ 64px testable), séparés par des séparateurs `rgba(255,255,255,0.08)`.
- Ligne file d'attente : prochains numéros en défilement doux, **64px**.
- Header : logo tenant, label "APPELS EN COURS", horloge temps réel — hauteur 64px, discret.

**UBIQUITAIRE** : Tous les tokens de couleur, taille et espacement sont lus depuis les variables CSS du Design System ; aucune valeur en dur dans le code.

**QUAND** l'agence est vide de tickets appelés, l'écran doit afficher un état empty lisible à 10 mètres ("Aucun appel en cours") sans modifier la structure de la page.

**QUAND** le socket WS est déconnecté, l'écran doit afficher un bandeau discret "Hors ligne — reconnexion…" sans masquer le dernier état connu.

**SI** la couleur `--brand` du tenant ne passe pas un ratio de contraste ≥ 4,5:1 sur `--surface-screen`, ALORS le système la fonce automatiquement (contraste auto-corrigé).

**LÀ OÙ** `prefers-reduced-motion` est actif, toutes les transitions doivent être désactivées.

**Anormal** : SI le payload `ticket:called` ne valide pas le schéma Zod du contrat, l'événement doit être ignoré et loggé ; l'affichage ne change pas.
