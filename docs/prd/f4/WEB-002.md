## WEB-002 : Interface agent — 3 boutons fixes, ticket 96px + chrono, raccourci Espace

**Agent** : agent-web · **Module** : F4 · **Dépend de** : WEB-001 · **Statut** : TODO

**Révision** : v2 — arbitrage 19

> DESIGN-gate humain obligatoire avant implémentation (PRD §4, règle 5).
> Code contre le mock Prism + événements simulés — sockets réels = RT-001.

---

### Exigences (EARS)

**UBIQUITAIRE** : L'interface agent doit présenter en permanence les 3 actions dans un ordre fixe et mémorisable :
1. **APPELER LE SUIVANT** — pleine largeur, couleur `--brand`, 88px de hauteur ; espace clavier déclenche cet appel quand le focus est sur la page.
2. **TERMINER** — affiché uniquement quand un ticket est en `SERVING`, largeur 50%.
3. **TRANSFÉRER** — affiché uniquement quand un ticket est en `SERVING`, largeur 50%.
Aucun autre bouton, aucun menu caché (anti-pattern Design System §13).

**UBIQUITAIRE** : Le ticket en cours doit s'afficher en **96px** (token `kpi-value`), police "Clash Display", couleur `--brand`. Le chronomètre s'incrémente à chaque seconde, format `MM:SS`, typographie Mono `--caption`.

**QUAND** l'agent clique "APPELER LE SUIVANT" (ou presse Espace), l'interface doit :
1. Envoyer `POST /api/v1/counters/{counterId}/call-next` via le client typé généré depuis `core.yaml` (route canonique — `/call` ciblée n'est pas dans l'UI agent).
2. Passer en état loading sur le bouton (spinner, disabled) pendant la réponse.
3. À réception du payload `ticket:called` du WS (ou de la réponse HTTP), afficher le nouveau ticket et démarrer le chrono.

**QUAND** la file est vide et l'agent clique "APPELER LE SUIVANT", l'interface doit afficher "Aucun client en attente" sans alerte (comportement normal, Design System §12).

**QUAND** l'agent clique "TERMINER", l'interface doit envoyer `PATCH /tickets/{id}/close` et réinitialiser la zone ticket (état empty).

**QUAND** l'agent clique "TRANSFÉRER", l'interface doit ouvrir un sélecteur de guichet/service de destination (composant inline, pas de modale — règle Design System §13).

**SI** la connexion WS est perdue, le chrono continue localement et un bandeau "Hors ligne" s'affiche ; les actions sont maintenues et rejoindront la queue de resync à la reconnexion.

**LÀ OÙ** le rôle n'est pas `AGENT`, la route `/agent` retourne 403 (contrôle hérité WEB-001).
