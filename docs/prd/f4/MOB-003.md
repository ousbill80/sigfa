## MOB-003 : Ticket vivant — position temps réel, Live Activity iOS / notification persistante Android

**Module** : F4 · **Agent** : agent-mobile · **Dépend de** : MOB-002 · **Statut** : TODO

**Révision** : v2 — arbitrage 19

---

### Exigences (EARS)

- QUAND un ticket est actif (statut `WAITING` ou `CALLED`), l'écran Ticket Vivant doit occuper l'intégralité de l'écran avec une carte affichant : `displayNumber`, position dans la file, barre de progression vers "c'est votre tour", `estimatedWaitMinutes` mis à jour toutes les 30 secondes (polling `GET /public/tickets/{trackingId}` + `ETag` pour économiser la bande passante).
- QUAND l'événement Socket.io `queue:updated` est reçu (payload : `queueId`, `length`, `estimate`), la carte doit se mettre à jour instantanément sans recharger l'écran — la connexion WS émet `sync:request` à la reconnexion. **En F4 : mock WS local derrière flag `EXPO_PUBLIC_MOCK_WS=true` (tests d'événements) — AUCUNE connexion socket réelle avant RT-001.**
- QUAND l'événement `ticket:called` est reçu, la carte doit passer à l'état "Votre tour !" avec animation brand 400ms (fallback statique si `prefers-reduced-motion`).
- Sur iOS : **Live Activity** (ActivityKit) affichant position + estimation sans ouvrir l'app — structure déclarée **derrière flag `EXPO_PUBLIC_LIVE_ACTIVITY=false`** en F4 ; critère CI = pas de crash flag off ; activation EAS = story post-F4. Test = pas de crash quand flag off.
- Sur Android, le système doit afficher une **notification persistante** (canal `TICKET_ACTIVE`, priorité HIGH, non dismissable) avec position + estimation — elle disparaît à la clôture.
- Le système doit gérer **5 états CANONIQUES distincts** : nominal (ticket en attente), called (votre tour), done (service terminé), **offline** (dernière position connue + badge discret — skeleton distinct du loading), **error** (ticket introuvable → message "Adressez-vous à l'accueil") — `loading` (skeleton) et `empty` sont distincts.
- LÀ OÙ la géolocalisation est accordée, la carte doit afficher le temps de trajet estimé vers l'agence (optionnel, non bloquant si refusé).
