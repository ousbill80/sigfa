### Exigences (EARS)

**UBIQUITAIRE** : La console admin doit exposer les sections suivantes, chacune contrainte par RBAC (matrice MODULE 4) :
- **Identité banque** (BANK_ADMIN+) : logo upload (R2), nom affiché, couleur `--brand` avec preview contraste auto-corrigé en temps réel.
- **Gestion agences** (BANK_ADMIN+) : CRUD agences, horaires par jour, jours fériés ivoiriens pré-intégrés, fermetures exceptionnelles.
- **Catalogue services** (BANK_ADMIN+, AGENCY_DIRECTOR sur son agence) : CRUD services, code, SLA en minutes, priorité, actif/inactif.
- **Gestion guichets** (AGENCY_DIRECTOR+) : CRUD guichets numérotés, affectation service.
- **Gestion agents** (AGENCY_DIRECTOR+) : CRUD users rôle AGENT, import CSV (`POST /api/v1/agencies/{id}/agents/import`).
- **Templates SMS** (BANK_ADMIN+) : éditeur de templates par événement (confirmation, "vous êtes 3e", "vous êtes le suivant"), variables `{{ticket}}` `{{position}}` `{{estimate}}` `{{agency}}`.
- **Seuils d'alerte** (MANAGER+) : file critique (nb tickets), inactivité agent (minutes), configurable par agence.
- **Onboarding agence** : parcours guidé 5 étapes (création → template → services → guichets → agents → QR d'installation) ; la durée totale cible est < 2 heures.

**UBIQUITAIRE** : Tous les appels API sont issus du client typé généré depuis `admin.yaml`. Chaque mutation utilise le header `X-Idempotency-Key` (UUID v4 généré côté client) pour l'idempotence.

**QUAND** l'utilisateur clique "Sauvegarder", le formulaire valide via Zod côté client (schémas `@sigfa/schemas`) avant d'envoyer la requête ; les erreurs de validation s'affichent en ligne sans modale.

**QUAND** l'import CSV agents se termine, la console affiche un résumé : N créés, M ignorés (doublons), K erreurs avec motif par ligne.

**QUAND** la couleur `--brand` saisie par le BANK_ADMIN ne passe pas le ratio 4,5:1 sur `--surface-1`, un avertissement inline indique la valeur corrigée appliquée automatiquement.

**SI** une agence est désactivée pendant que des tickets sont ouverts, un dialog de confirmation liste les tickets concernés ; l'agent peut annuler ou confirmer la fermeture forcée.

**LÀ OÙ** le rôle est `AGENT` ou `AUDITOR`, toutes les routes `/admin/*` retournent 403.

**Anormal** : SI l'API `admin.yaml` retourne 409 (conflit), le message d'erreur traduit le code métier en langage naturel ("Ce code de service existe déjà dans cette agence").
