### Exigences (EARS)

**UBIQUITAIRE** : Le dashboard réseau doit afficher en temps réel, scope multi-agences de la banque du claim JWT :
- **Classement agences** : liste triée par TMA décroissant, badge coloré par seuil (`--success` si TMA ≤ SLA, `--warning` si TMA ≤ 2× SLA, `--danger` si TMA > 2× SLA). `--danger` réservé aux agences en dépassement — jamais décoratif.
- **Carte géographique** : marqueurs positionnés sur `agency.city` avec la même couleur de badge. Implémentation : **SVG statique Côte d'Ivoire** inscrit dans le dépôt CI (Leaflet exclu — zéro dépendance externe, offline-friendly).
- **Panneau alertes** : agrège les `alert:manager` de toutes les agences de la banque ; chaque alerte indique l'agence source.

**UBIQUITAIRE** : Les données sont issues de `GET /api/v1/reports/benchmark` et `GET /api/v1/admin/network-overview` (contrat `reporting.yaml` LA LOI — la route /reports/network est rejetée comme invention) agrégées par `bankId`. Mise à jour temps réel via `agency:offline` et `alert:manager`.

**QUAND** un événement `agency:offline` est reçu, le marqueur carte et la ligne classement de l'agence doivent passer en état dégradé (icône hors ligne, badge `--info`).

**QUAND** la banque a plus de 20 agences, le classement doit être paginé (20 par page) avec tri conservé.

**LÀ OÙ** le rôle est `BANK_ADMIN` ou `AGENCY_DIRECTOR`, seules les agences de leur périmètre JWT sont visibles.

**LÀ OÙ** le rôle est `AGENT`, la route `/dashboard/network` retourne 403.

**SI** aucune agence n'est disponible (état empty), un message guide vers WEB-006 pour configurer la première agence.
