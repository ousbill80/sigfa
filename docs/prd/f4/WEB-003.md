### Exigences (EARS)

**UBIQUITAIRE** : Le dashboard doit respecter la **hiérarchie en Z** définie dans le Design System §9 :
- Quadrant haut-gauche : TMA temps réel en **40px** (`kpi-value`) coloré `--success` / `--warning` / `--danger` selon le ratio vs SLA configuré — **`--danger` utilisé uniquement si dépassement SLA**, jamais décoratif.
- Quadrant haut-droit : taux d'abandon + NPS du jour.
- Zone médiane : file par service (nb tickets en attente + nb agents actifs + estimation), mise à jour via `queue:updated`.
- Zone basse : grille agents avec statut temps réel (`counter:status`) — nom, guichet, statut coloré, ticket en cours.
- Sous chaque KPI : sparkline Recharts 24h (mini-graphe sans axes, trait `--brand`, fond transparent).
- Bandeau comparatif discret J-7 sous le TMA.

**UBIQUITAIRE** : Tous les KPIs sont issus du client typé généré depuis `reporting.yaml` — route canonique : `GET /api/v1/reports/kpis?scope=agency` (LA LOI — les alias /reports/live sont rejetés) — mis à jour via les événements WS `queue:updated`, `counter:status`, `alert:manager`.

**QUAND** un événement `alert:manager` de type `SLA_BREACH` est reçu, le système doit afficher une card d'alerte persistante en `--danger` dans un panneau dédié ; la card disparaît sur acquittement manuel du manager.

**QUAND** un événement `alert:manager` de type `AGENT_INACTIVE` ou `AGENT_DISCONNECTED_WITH_TICKET` est reçu, le statut de l'agent concerné dans la grille passe en rouge avec une icône d'alerte.

**QUAND** le socket WS est déconnecté, le dashboard doit afficher un badge "Hors ligne — dernière sync [HH:MM]" et figer les KPIs sur le dernier état connu ; à la reconnexion, émettre `sync:request` et rafraîchir.

**QUAND** le rôle est `MANAGER`, la grille agents doit afficher une action inline par guichet pour basculer le statut OPEN/PAUSED via `PATCH /api/v1/counters/:id` (parcours P3 complet — composant inline, pas de modale, Design System §13) ; l'état est reflété en temps réel via `counter:status`.

**LÀ OÙ** le rôle est `AUDITOR`, le dashboard est accessible en lecture seule (aucun bouton d'action affiché).

**LÀ OÙ** le rôle est `AGENCY_DIRECTOR` ou `MANAGER`, les données sont filtrées sur l'agence du claim JWT.

**SI** aucune donnée n'est disponible (état empty), un message contextuel guide le manager sans KPI à zéro trompeur.
