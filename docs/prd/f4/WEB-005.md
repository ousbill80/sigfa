### Exigences (EARS)

**UBIQUITAIRE** : Le dashboard COMEX doit afficher **exactement 3 KPIs** (aucun de plus, règle Design System "une information principale") :
1. **NPS global réseau** — valeur en `40px` (`kpi-value`), delta vs mois précédent.
2. **TMA moyen réseau** — valeur en `40px`, coloré `--success` / `--warning` / `--danger` selon SLA réseau.
3. **Volume clients servis** — valeur en `40px`, mois en cours vs mois précédent (delta %).
Source : `GET /api/v1/reports/kpis?scope=network` (contrat `reporting.yaml` LA LOI — la route /reports/comex est rejetée comme invention) — exactement **3 KPIs** exploités : NPS global, TMA moyen, Volume clients servis.

**UBIQUITAIRE** : Le **mode TV** est activé par un paramètre d'URL `?tv=1` ou un toggle accessible uniquement aux rôles `BANK_ADMIN` et supérieurs. En mode TV :
- Toute la typographie est multipliée par **1,5** (token `--scale-tv: 1.5`) via une classe CSS racine.
- Les marges et espacements sont multipliés par 1,5.
- Les boutons et contrôles interactifs sont masqués (l'écran est en lecture seule projeté).
- La mise à jour des KPIs reste temps réel (WS actif).

**QUAND** le mode TV est actif et que le socket WS se déconnecte, l'écran doit afficher un bandeau discrèt "Hors ligne" sans masquer les KPIs.

**QUAND** les données du mois sont incomplètes (1er du mois, peu de tickets), les KPIs doivent afficher la valeur disponible avec une annotation "données partielles" — jamais de 0 brut sans contexte.

**LÀ OÙ** le rôle est `AGENT`, `MANAGER` ou `AGENCY_DIRECTOR`, la route `/dashboard/comex` retourne 403.

**LÀ OÙ** `prefers-reduced-motion` est actif, les transitions de mise à jour des KPIs sont instantanées.

**SI** le NPS est négatif (< 0), la valeur s'affiche en `--danger` ; `--danger` reste réservé aux alertes, pas utilisé sur les KPIs neutres.
