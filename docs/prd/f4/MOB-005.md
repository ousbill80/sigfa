## MOB-005 : Feedback post-service + historique de tickets

**Module** : F4 · **Agent** : agent-mobile · **Dépend de** : MOB-003 · **Statut** : TODO

**Révision** : v2 — arbitrage 19

---

### Exigences (EARS)

- QUAND le ticket passe à l'état `DONE`, le système doit afficher automatiquement l'écran de feedback (note 1–5 étoiles + commentaire optionnel ≤ 500 caractères) dans la fenêtre de 24h après clôture.
- Le système doit appeler `POST /public/tickets/{trackingId}/feedback` avec la note et le commentaire — `trackingId` au format nanoid(21) ; un deuxième envoi retourne 409 `FEEDBACK_ALREADY_SUBMITTED` (affiché en message non bloquant).
- SI la fenêtre de 24h est expirée (`FEEDBACK_WINDOW_EXPIRED`), ALORS l'écran de feedback est masqué et remplacé par la vue historique sans message culpabilisant.
- Le registre de copie SIGFA s'applique : le bouton dit "Donner mon avis" (pas "Soumettre") ; le message de confirmation est "Merci pour votre retour !" ; le commentaire est labellisé "Un mot à ajouter ? (facultatif)".
- L'écran Historique doit lister les tickets précédents stockés en MMKV (trackingId, displayNumber, date, statut, note si donnée) — paginé localement, infini scroll, vide si aucun ticket.
- LÀ OÙ l'utilisateur n'a pas encore donné son avis sur un ticket DONE dans la fenêtre, un badge de rappel discret doit apparaître sur l'onglet Historique (jamais de notification push pour le feedback).
- Le système doit gérer 5 états : nominal (formulaire actif), loading (envoi en cours), empty (aucun historique), error (réseau KO → retry depuis MMKV), success (confirmation post-envoi).
