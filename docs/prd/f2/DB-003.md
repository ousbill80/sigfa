# DB-003 : Migrations initiales consolidées + seed — services par défaut, rôles, jours fériés CI

**Module** : F2 — Data · **Agent** : agent-database · **Dépend de** : DB-002 · **Statut** : TODO

## Exigences (EARS)

- Le seed (`src/seed/`, exécutable `pnpm --filter @sigfa/database seed`) doit être **idempotent** : QUAND il est exécuté deux fois, l'état final est identique (upsert par clés naturelles, zéro doublon).
- Le seed doit créer le **catalogue de services par défaut** (gabarit clonable au niveau banque, v5 §MODULE 1) : OC 15 min, OA 30, CR 45, CH 10, EN 45, VIP 20, RE 30, EP 25 — codes et SLA exacts.
- Le seed doit créer les **jours fériés ivoiriens** (table `public_holidays`, sans bank_id — référentiel national) : fêtes fixes + mobiles 2026-2027 pré-remplies avec **`is_approximate` boolean** sur les dates estimées, sources dans `src/seed/public-holidays-sources.md` ; **GRANT SELECT uniquement au rôle applicatif** (REVOKE INSERT/UPDATE/DELETE — exception RLS documentée) ; SI l'année courante dépasse `max(year)` des fériés mobiles, ALORS un avertissement doit être loggé au seed/démarrage (mécanisme de mise à jour annuelle documenté — story d'exploitation consignée).
- Le seed doit créer un **tenant de démonstration** complet derrière un flag explicite (`SEED_DEMO=1`) : 1 banque, 2 agences, services actifs, 2 guichets, 1 kiosque, comptes utilisateur par rôle (mots de passe forts générés, affichés une fois) — JAMAIS en production par défaut.
- Les rôles étant une enum (pas une table), la story doit fournir la **matrice RBAC versionnée** consommable par l'API : `src/seed/rbac-matrix.ts` exporté (source : v5 §MODULE 4), avec test d'exhaustivité (6 rôles × actions).
- La chaîne de migrations 001+002 doit être consolidée et rejouable : PG16 vierge → `migrate` → `seed` → base opérationnelle.
