# DB-007 : Tables IA — prédictions, anomalies, recommandations, scores + rétention

**Module** : F2 — Data · **Agent** : agent-database · **Dépend de** : DB-008, CONTRACT-008 · **Statut** : TODO

## Exigences (EARS)

- `ai_forecasts` : `bank_id`, `agency_id`, `target_date`, `hour`, `expected_tickets`, `confidence` (0–1), `factors` jsonb (enum facteurs LA LOI), `model_version`, `computed_at`, `data_window` — unique `(bank_id, agency_id, target_date, hour, model_version)` ; les métadonnées portent AiMeta de LA LOI.
- `ai_staffing_recommendations` : `bank_id`, `agency_id`, `target_date`, `time`, `action`, `counters`, `rationale`, `status` (PENDING/ACKED), `acked_by?`, `acked_at?`.
- `ai_anomalies` : `bank_id`, `agency_id?`, `type` (enum LA LOI : QUEUE_STUCK/AGENT_INACTIVE_PATTERN/SLA_SYSTEMIC), `status` (open/acked/resolved), `payload` jsonb, `detected_at`, `acked_by?/acked_at?/resolved_at?` — index `(bank_id, status, detected_at)`.
- `ai_quality_scores` : `bank_id`, `agency_id`, `agent_id?`, `period` date, `score`, `components` jsonb, `model_version` — agrégats anonymisés uniquement (aucun verbatim client).
- **Rétention IA** : les prédictions/scores de plus de 24 mois doivent être purgeables (`purgeAiHistory()` idempotente, alignée sur la politique DB-008) ; les anomalies résolues >24 mois aussi.
- UBIQUITAIRE : aucune donnée personnelle (téléphone, verbatim) dans ces tables ; entraînement sur données du tenant uniquement (frontière logique — documentée, enforcée en F10).
