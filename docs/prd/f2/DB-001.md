# DB-001 : Schéma cœur Drizzle — tenants, agences, services, files, guichets, kiosques, utilisateurs, tickets

**Module** : F2 — Data · **Agent** : agent-database · **Dépend de** : F1 DONE + **CONTRACT-011 DONE** (TicketPriority/Service.code) · **Statut** : TODO
**Révision** : v2 — amendée après critique (13 BLOCKER intégrés)

## Exigences (EARS)

### Tables tenants & structure
- `banks` : `slug` unique, **`theme` jsonb (colonne UNIQUE alignée BankTheme de LA LOI : requestedColors, appliedColors, welcomeMessages, logoUrl — pas de colonne colors séparée)**, **seuils bornés CHECK** : `queue_critical_threshold int NOT NULL DEFAULT 50 CHECK 1..500`, `agent_inactivity_minutes int NOT NULL DEFAULT 15 CHECK 1..60`, `no_show_timeout_minutes int NOT NULL DEFAULT 3 CHECK 1..30`.
- `agencies` : ville, timezone défaut Africa/Abidjan, **`weekly_schedule` jsonb** (7 jours {start,end}), **`is_template` boolean NOT NULL DEFAULT false**, is_active + table **`agency_exceptional_closures`** (`bank_id`, `agency_id`, `date`, `reason`) — les fermetures n'écrasent jamais l'hebdo.
- `services` : `code` varchar(4) `^[A-Z]{2,4}$` unique par agence (LA LOI post-011), `name`, `sla_minutes`, **`display_order` int NOT NULL DEFAULT 0** (aligné `order` du contrat — PAS de colonne priority), `is_active`.
- `queues` : service_id, `current_ticket_number int NOT NULL DEFAULT 0 CHECK (>=0)` (incrément par l'API en `UPDATE … RETURNING` lock-then-increment — documenté en commentaire, logique API-003), `is_open`.
- `counters` : number, label, status enum, agent_id?, current_ticket_id? + table **`counter_services`** (n-n, unique (counter_id, service_id)) — support routage API-004.
- `kiosks` : label, **`credentials_hash` bcrypt cost 12**, last_seen, printer_status enum, app_version, **`current_session_id uuid?`, `session_expires_at?`, `session_revoked_at?`** (révocation auditée en base ; token en Redis).

### Utilisateurs & agents
- `users` : **`bank_id uuid NULL` + `CHECK ((role = 'SUPER_ADMIN') = (bank_id IS NULL))`** (précision post-implémentation : tout utilisateur est rattaché à sa banque SAUF SUPER_ADMIN plateforme — la RLS de DB-002 s'applique sur users), `email` **UNIQUE GLOBAL** (login sans bankId ; multi-banque même email = story future), password_hash (bcrypt 12), first/last_name, role enum (6 valeurs), **`languages text[] NOT NULL DEFAULT '{FR}'`** (valeurs FR/DIOULA/BAOULE/EN), **`work_schedule jsonb?`**, **`failed_login_attempts int NOT NULL DEFAULT 0`**, **`locked_until timestamptz?`** (API-001 : 5 tentatives → +15 min), `phone_encrypted text?` + **`phone_hash text?`**, is_active.
- **`user_services`** (n-n user↔service, unique) — compétences ; **`agency_users`** (n-n user↔agency).
- **`agent_status_history`** : `bank_id`, `agency_id`, `agent_id`, `from_status`, `to_status` (enum AgentStatus), `changed_at` — index `(bank_id, agent_id, changed_at)` — source d'`agent_active_seconds` (DB-006) et des patterns d'anomalies (F10).

### Tickets
- `tickets` : bank_id, agency_id, queue_id, service_id, counter_id?, agent_id?, `number` int, `display_number` text **format `{code}-{NNN}`** (ex. OC-047, composé par l'API), **`tracking_id` char(21) unique** (nanoid), **`local_uuid` uuid unique?** (idempotence sync), `channel` enum, `status` enum, **`priority` enum TicketPriority (LA LOI post-011) NOT NULL DEFAULT 'STANDARD'**, `phone_encrypted text?` + **`phone_hash text?` index `(bank_id, phone_hash)`**, `sms_consent` bool défaut false, horodatages issued/called/served/closed, wait/service_time_seconds?, feedback (score 1–5, comment ≤500, at), no_show_at?.
- **`issued_day date GENERATED ALWAYS AS ((issued_at AT TIME ZONE 'Africa/Abidjan')::date) STORED`** + contrainte unique `(queue_id, number, issued_day)`.
- **`ticket_transfers`** : bank_id, ticket_id, from_counter_id?, from_service_id, to_service_id, to_counter_id?, reason?, transferred_by, transferred_at — historique des cascades.

### Transverse
- CHAQUE table métier (services, queues, counters, kiosks, agency_users, user_services, transfers, history incluses) : `bank_id uuid NOT NULL`, index composites bank_id en tête ; enums = LA LOI (exception Role : sous-ensemble strict \ {NONE}, testé + NONE absent de pg_enum) ; drizzle-kit strict ; FK RESTRICT ; conventions `_dag.md`.
