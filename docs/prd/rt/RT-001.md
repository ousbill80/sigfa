## RT-001a — Enablement serveur (agent-api, `apps/api/**`)

### Exigences (EARS)
- **Bascule autoritaire** : `REALTIME_MODE ∈ {off, real}` (défaut `off`, y compris tous les tests). **SI** `off` **ALORS** le bootstrap garde `createNoopBus`, n'ouvre ni socket ni scheduler (zéro régression des suites F3). **SI** `real` **ALORS** socket + bus réel + scheduler.
- **Ordre de bootstrap** (UBIQUITAIRE, `index.ts::startServer`, sous `real`) : connect PG/Redis (fail-fast) → `serve()` capture le `httpServer` (cast `as unknown as http.Server`, retour union `ServerType`) → `createSocketServer(httpServer, {db,redis,jwtSecret})` → `startAlertScheduler` (jobs BullMQ `inactive-agent-scan`/`sla-scan`, verrou distribué, concurrency=1).
- **Graceful shutdown** (QUAND SIGTERM/SIGINT) : `io.close()` → fermeture des workers BullMQ (drain borné, timeout documenté) → fermeture PG/Redis. Aucun worker/socket orphelin.
- **Signature de bus étendue** : `RealtimeBus.emit(event, agencyId, payload)`. Les sites d'émission F3 (`tickets.ts`, `tickets-sync.ts`, `queues.ts`, `agent-status.ts`, `agent-disconnect.ts`) passent l'`agencyId` (dérivé du scope tenant/route ; résolu une fois là où absent — `counter:status`/`alert:manager`). Les tests F3 de ces sites sont mis à jour (CaptureBus enregistre `(event, agencyId, payload)`).
- **`createSocketBus(io)` = ADAPTATEUR contrat** : `emit(event, agencyId, payload)` (1) MAPPE la forme interne → **forme du CONTRAT** (`packages/contracts/events/realtime.ts`) par événement (hydratation des champs manquants ; lookup DB minimal seulement si un champ requis est absent, hors chemin latence), (2) valide la sortie contre le `payloadSchema` du **contrat**, (3) diffuse `io.to('agency:'+agencyId).emit(event, contractPayload)`. **`emitTicketCalled` est absorbé** (chemin d'émission unique — fin de la double forme `ticket:called`).
- **anormal** : payload invalide (post-mapping) → NON diffusé + log, sans impact sur les émissions valides.
