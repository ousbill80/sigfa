# ADM-003 : Supervision bornes — statut ping temps réel, alertes « borne muette »

**Module** : F8 — ADMIN & THEMING · **Dépend de** : API-011, WEB-004 · **Statut** : TODO

**Révision** : v1 — expansion Boucle 1 (Fable 5)

> **But** : voir en un coup d'œil l'état de santé de **toutes les bornes** d'un périmètre (agence → réseau), et être alerté quand une borne **cesse de répondre** (« borne muette »).

> **Définition — « borne muette »** : une borne enrôlée dont le **dernier heartbeat reçu remonte à plus que le seuil de silence**. Modèle explicite :
> - Une borne saine émet un **heartbeat toutes les `H` secondes** (défaut `H = 30 s`).
> - `ONLINE` : dernier heartbeat < `2 × H` (défaut < 60 s).
> - `DEGRADED` : dernier heartbeat ∈ [`2 × H`, `seuil_muette`) — retards/pertes intermittentes.
> - `SILENT` (**muette**) : dernier heartbeat ≥ **`seuil_muette` = 3 heartbeats manqués** (défaut **90 s**), configurable par agence. C'est ce passage à `SILENT` qui déclenche l'alerte.
> - `NEVER_SEEN` : borne provisionnée (ADM-002) mais jamais enrôlée/vue.

> **Découpage (Loi 4)** : ADM-003a (agent-api — heartbeat, calcul d'état, alertes) et ADM-003b (agent-web — écran de supervision). Front sur mock Prism ; temps réel réel = RT-001.

## ADM-003a — API : heartbeat, machine d'état borne, alerte « muette » (agent-api)

**Agent** : agent-api · **Dépend de** : API-011

### Exigences (EARS)

**UBIQUITAIRE** : L'API doit exposer sous `admin.yaml` :
- `POST /api/v1/kiosks/{kioskId}/heartbeat` (auth borne) → corps `{ ts, fw?, printerOk?, queueDepth? }` → enregistre le dernier `seenAt` et métadonnées légères ; retourne 204.
- `GET /api/v1/agencies/{agencyId}/kiosks/status` (MANAGER+) → liste `[{ kioskId, label, status, lastSeenAt, staleForSeconds }]`.
- `GET /api/v1/banks/{bankId}/kiosks/status` (BANK_ADMIN+, direction réseau) → agrégat réseau `{ online, degraded, silent, neverSeen, byAgency[] }`.

**UBIQUITAIRE** : L'état d'une borne est **dérivé du temps écoulé depuis `lastSeenAt`** selon la définition ci-dessus (`ONLINE`/`DEGRADED`/`SILENT`/`NEVER_SEEN`), avec `H` et `seuil_muette` configurables par agence (défauts 30 s / 90 s). Le calcul est **centralisé et déterministe** (pur, testable sans horloge réelle — horloge injectée).

**QUAND** l'état d'une borne bascule vers `SILENT`, le système doit émettre l'événement temps réel `kiosk:silent` (conforme à un ajout CONTRACT-002 — voir `_notes.md`) et déclencher une **alerte** (persistée + notifiable au manager via le canal alertes existant, NOTIF/manager). Le basculement est **débounced** : une seule alerte par épisode de silence (pas de tempête).

**QUAND** une borne muette **réémet** un heartbeat, le système repasse son état à `ONLINE`, **résout** l'alerte de silence (`resolvedAt`), et émet `kiosk:recovered`.

**QUAND** un client interroge `.../status`, l'état est calculé **à la lecture** à partir de `lastSeenAt` et de l'horloge serveur (jamais un état figé potentiellement périmé).

**SI** un heartbeat arrive pour un `kioskId` d'un autre tenant → 403 opaque (tenant-isolation). SI le `kioskId` est inconnu → 404 opaque.

**LÀ OÙ** une borne est `NEVER_SEEN` depuis plus d'un délai de grâce post-provision (défaut : > 24 h après `provisionedAt`), elle est signalée distinctement (« installation non finalisée ») sans être comptée comme panne.

**Anormal** : heartbeat avec `ts` dans le futur ou trop ancien (> skew toléré) → accepté mais `seenAt` = horloge serveur (jamais l'horloge borne pour le calcul d'état).

### Critères d'acceptation

- [ ] `ADM-003a: heartbeat met à jour lastSeenAt ; 204 ; métadonnées légères persistées`
- [ ] `ADM-003a: état ONLINE/DEGRADED/SILENT/NEVER_SEEN dérivé du délai (horloge injectée, table de cas)`
- [ ] `ADM-003a: seuil_muette = 3 heartbeats manqués (défaut 90s), configurable par agence`
- [ ] `ADM-003a: bascule vers SILENT → événement kiosk:silent + alerte persistée (une seule par épisode)`
- [ ] `ADM-003a: heartbeat après silence → ONLINE + alerte resolvedAt + kiosk:recovered`
- [ ] `ADM-003a: status calculé à la lecture depuis lastSeenAt (jamais figé)`
- [ ] `ADM-003a: agrégat réseau online/degraded/silent/neverSeen + byAgency`
- [ ] `ADM-003a: heartbeat autre tenant → 403 opaque ; kioskId inconnu → 404 (tenant-isolation)`
- [ ] `ADM-003a: NEVER_SEEN > 24h post-provision → « installation non finalisée », pas comptée en panne`

## ADM-003b — Web : écran de supervision des bornes (agent-web)

**Agent** : agent-web · **Dépend de** : WEB-004, ADM-003a

### Exigences (EARS)

**UBIQUITAIRE** : La console doit fournir un **écran de supervision des bornes** :
- Vue **agence** (MANAGER+) : grille/liste des bornes de l'agence avec `StatusPill` (Design System v2 §4) — `ONLINE` (=`--success`/forest), `DEGRADED` (=`--warning`), `SILENT` (pastille/pictogramme `--danger`, **jamais un fond rouge plein** — Design System v2 §1), `NEVER_SEEN` (neutre + libellé « jamais vue »), avec `lastSeenAt` relatif (« il y a 12 s »).
- Vue **réseau** (BANK_ADMIN+, prolonge WEB-004 direction réseau) : compteurs online/degraded/silent + liste des agences ayant ≥ 1 borne muette, triées par gravité.

**UBIQUITAIRE** : L'écran s'actualise en **temps réel** via `SocketProvider` (événements `kiosk:silent`/`kiosk:recovered`) ; en l'absence de socket (mock), il **poll** `.../status` à intervalle court. À la reconnexion WS → resync complet de l'état.

**QUAND** une borne devient muette, sa carte passe visiblement en état d'alerte (pastille danger + libellé « Borne muette — dernier signe il y a {X} ») et remonte en tête de liste ; un compteur d'alertes actives est visible.

**QUAND** une borne muette redevient active, sa carte revient à `ONLINE` et l'alerte disparaît de la liste active (consultable en historique).

**SI** aucune borne n'existe encore (nouvelle agence) → `EmptyState` « Aucune borne enregistrée — provisionnez depuis l'onboarding » (lien ADM-002), pas un tableau vide anxiogène.

**LÀ OÙ** le rôle est `AGENT` ou `AUDITOR`, l'écran de supervision retourne 403 (RBAC MODULE 4).

**Anormal** : SI l'API status est indisponible, bandeau `--info` « Supervision indisponible — dernière mise à jour il y a {X} » et affichage du dernier état connu (jamais un écran vide sans contexte).

### Critères d'acceptation

- [ ] `ADM-003b: vue agence — StatusPill ONLINE/DEGRADED/SILENT/NEVER_SEEN + lastSeenAt relatif`
- [ ] `ADM-003b: SILENT rendu en pastille/pictogramme danger, jamais fond rouge plein (axe-core + snapshot)`
- [ ] `ADM-003b: vue réseau — compteurs + agences avec bornes muettes triées par gravité`
- [ ] `ADM-003b: temps réel — kiosk:silent → carte en alerte + remontée en tête + compteur`
- [ ] `ADM-003b: kiosk:recovered → retour ONLINE, alerte hors liste active (historique consultable)`
- [ ] `ADM-003b: reconnexion WS → resync complet de l'état`
- [ ] `ADM-003b: aucune borne → EmptyState avec lien onboarding (jamais tableau vide)`
- [ ] `ADM-003b: RBAC AGENT/AUDITOR → 403`
- [ ] `ADM-003b: API status indisponible → bandeau info + dernier état connu`
- [ ] `ADM-003b: états loading (skeleton grille) / offline`
- [ ] `ADM-003b: tokens uniquement — axe-core PASS`
