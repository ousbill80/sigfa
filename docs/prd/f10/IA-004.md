# IA-004 : NLP feedbacks (FR/EN) + scoring qualité auto par agence/agent

**Module** : F10 — IA & Prédiction · **Agent** : agent-api · **Dépend de** : API-010 (feedback client note+commentaire), IA-001 (pipeline features / jointure scoring) · **Statut** : PRD PRÊT — **implémentation GATED sur données réelles de production**

> **⚠️ Gating produit** : un scoring qualité fiable exige un volume réel de feedbacks. **Implémentation runtime BLOQUÉE** sur données de production. Le **NLP lui-même** (classification sentiment/thème) est **évaluable AVANT production** sur un **corpus étiqueté synthétique/annoté FR-EN** (critères ⊛). Le seuil de volume minimal par agence/agent avant publication d'un score est gated. Voir `_notes.md`.

> **⚠️ ÉCART CATALOGUE CORRIGÉ — LANGUES** : le catalogue F10 mentionne « FR + **langues locales** ». **Décision PO prioritaire : FR/EN UNIQUEMENT.** Dioula et Baoulé sont **RETIRÉS** (cf. mémoire projet « langues FR/EN only »). Le NLP de cette story traite **le français et l'anglais**, rien d'autre. Toute mention de « langues locales » du catalogue est **remplacée** par FR/EN. Cet écart est journalisé dans `_notes.md` (question PO : tranchée FR/EN).

## Exigences (EARS)

- **UBIQUITAIRE — périmètre linguistique FR/EN** : le pipeline NLP doit détecter la langue et traiter **uniquement FR et EN**. SI un feedback n'est ni FR ni EN (langue non supportée) ALORS il est marqué `language: unsupported` et **exclu du scoring** (jamais de classification hasardeuse), sans être perdu (conservé brut pour revue manuelle).
- **UBIQUITAIRE — source = feedbacks SIGFA du tenant** : l'analyse doit porter **uniquement** sur les feedbacks collectés par SIGFA (API-010 : note 1-5 + commentaire ; feedback borne KIOSK-009) de la banque, isolés par `bank_id`.
- **QUAND (WHEN) un commentaire FR/EN est analysé**, le système doit produire : `sentiment` (négatif/neutre/positif + score continu), `themes[]` (thèmes récurrents énumérés : attente, accueil, propreté, compétence, information, résolution…), et un `qualityScore` normalisé, le tout rattaché au scope `(agency, agent?, service?, période)`.
- **UBIQUITAIRE — scoring qualité agrégé** : le système doit calculer un **score qualité auto** par **agence** et par **agent**, combinant note structurée (API-010) et sentiment NLP, agrégé sur une période — surface exposée via `GET /ai/feedback-insights?period=&scope=` (CONTRACT-008). Les agrégats **réseau** héritent du schéma **`AnonymizedNetworkAggregate`** (reporting.yaml / CONTRACT-006, `$ref`, jamais redéfini).
- **UBIQUITAIRE — anti-PII (CRITIQUE)** : les **insights ne doivent contenir aucune donnée personnelle brute**. Le pipeline doit **redacter/anonymiser** avant tout stockage d'insight : aucun numéro de téléphone, nom, ou identifiant client dans les thèmes/citations exposés. Toute **citation** ("verbatim") remontée doit être **expurgée** des entités personnelles (masquage). Conforme CONTRACT-008 (« insights sans données personnelles brutes »).
- **SI (IF) le NLP est confié à un service tiers d'IA externe** (hébergement modèle hors infra SIGFA) **ALORS** cet envoi est **INTERDIT sans base légale UEMOA documentée** : par défaut, aucune donnée de feedback (même expurgée) ne quitte l'infra du tenant vers un service tiers. Toute exception exige une base légale + décision PO explicite (drapeau confidentialité `_notes.md`). **Par défaut : NLP hébergé dans l'infra SIGFA.**
- **SI (IF) le volume de feedbacks d'un agent/agence < seuil minimal (configurable, défaut à arrêter avec le PO) ALORS** aucun score individuel ne doit être **publié** (protection contre le jugement sur échantillon non significatif), état `INSUFFICIENT_SAMPLE`.
- **UBIQUITAIRE — pas de décision automatique** : le score qualité est **indicatif**. Aucune sanction, notation RH punitive automatique, ou action opérationnelle ne doit en découler automatiquement. L'humain interprète. Test explicite : aucune mutation hors table insights.
- **UBIQUITAIRE — explicabilité** : chaque score doit être décomposable (contribution note structurée vs sentiment, thèmes dominants) pour être justifiable auprès de l'agent concerné.
- **UBIQUITAIRE — métriques d'évaluation** : la qualité du classifieur doit être mesurée sur un **corpus FR/EN étiqueté** : **précision/rappel/F1** sentiment ET thèmes, **matrice de confusion**, avec suivi du **biais** (pas de dérive systématique de sentiment corrélée à une agence/langue). Un garde-fou vérifie la **parité de performance FR vs EN**.
