## Exigences (EARS)
- **Rate limiting global routes publiques** (Redis, fenêtre glissante) : /public/* et webhooks selon les valeurs documentées LA LOI (devices 10/min/IP…) — middleware unique paramétré par route, 429 conforme ; les limites par-route spécifiques (API-010) le réutilisent.
- `GET /health` : public, sans auth ni tenant — `{status, uptime, checks: {postgres, redis}}` en <100 ms, 503 si dépendance down (testé en coupant le conteneur).
- `POST /kiosks/:kioskId/heartbeat` (session borne) : met à jour last_seen/printer_status/app_version ; QUAND printer_status passe à ERROR/OFFLINE, émettre `kiosk:printer-error` (une fois par épisode) ; retour `{serverTime}`.
- `GET /kiosks/status` (MANAGER+) : liste LA LOI — statut ONLINE/SILENT dérivé : **SILENT si `last_seen < NOW() - 3 min`** (constante globale v1 : `KIOSK_HEARTBEAT_INTERVAL_S = 60`, seuil SILENT = 3× = 180 s ; exportée dans `config/kiosk.ts` ou `.env.example`).
- `GET /audit-logs` (SUPER_ADMIN|AUDITOR) : filtres/pagination LA LOI, mapping DB→API (occurred_at→timestamp, actor composé) — lecture seule stricte.
- `POST /notifications/devices` + DELETE : enregistrement idempotent (table DB-005), 429 dédié — le reste du module notifications = F6.
