## Exigences (EARS)
- `POST /agencies/:id/clone-from/:templateId` : copie CONFIG uniquement (services actifs, horaires, guichets+counter_services) — jamais de données ; source `is_template` recommandée (403 documenté sinon ? non : LA LOI ne l'exige pas — toute agence clonable, is_template = filtre UI ; consigné).
- **Session borne** : `POST /agencies/:id/kiosk-access` (crée kiosk + credentials bcrypt affichés une fois + QR d'installation) ; `POST /kiosk/session` (credentials → JWT 12 h scope agency role AUTHENTICATED, colonnes session sur kiosks) ; `DELETE /kiosk/session/:kioskId` (révocation → session_revoked_at, JWT refusé ensuite malgré exp valide).
- `POST /agents/import` : CSV LA LOI (colonnes fixées, ≤500, UTF-8) — rapport `{created, skipped, errors[{line,field,code,message}]}`, transaction par ligne, mots de passe initiaux aléatoires (jamais dans la réponse — envoi = F6, consigné), E.164 validé.
- **Theming** : `GET/PATCH /banks/:id/theme` — `appliedColors` = correction de contraste ≥4.5:1 (algorithme WCAG relative luminance, ajustement de lightness documenté, testé sur cas limites) ; `GET .../logo-upload-url` : URL signée R2 (S3 presign) — LÀ OÙ R2 n'est pas configuré (dev), 503 code documenté, jamais de crash.
- **Droit à l'oubli** : `POST /data/purge-phone` † (branche `purgePhone` DB-008, BANK_ADMIN, idempotent LA LOI) ; `GET /data/retention-policy`.
