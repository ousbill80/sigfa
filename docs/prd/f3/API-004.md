# API-004 : Moteur de file — priorités, routage compétence+langue, débordement, pause

**Module** : F3 — API · **Agent** : agent-api · **Dépend de** : API-003 · **Statut** : TODO

## Exigences (EARS)
- La sélection de `call-next` doit devenir : parmi les tickets WAITING des services du guichet (`counter_services`), choisir par **priorité décroissante VIP > PMR > SENIOR > PRIORITY > STANDARD** puis FIFO (issued_at) à priorité égale — pur, testable, remplace la stratégie API-003 par injection (`selectNextTicket(strategy)`).
- LÀ OÙ le ticket exige une langue (champ dérivé du canal/choix borne, nullable), SI l'agent du guichet ne la parle pas (users.languages), le ticket est sauté pour CE guichet (il reste éligible ailleurs) — règle documentée : la langue est une PRÉFÉRENCE de routage, jamais un blocage définitif (si aucun agent compatible, le premier guichet libre le prend après `LANGUAGE_SOFT_TIMEOUT_MINUTES` défaut 10, configurable).
- **Débordement** : SI la file d'un service dépasse `queue_critical_threshold` (banks), ALORS les guichets d'autres services COMPATIBLES (partageant au moins un agent qualifié) reçoivent aussi ses tickets en call-next + `alert:manager` QUEUE_CRITICAL (une fois par franchissement, pas en rafale).
- **Pause de file** : `PATCH /queues/:id` (is_open, plages LA LOI) — file fermée : émission → 422 code LA LOI, tickets existants restent servables ; réouverture → reprise.
- La position/estimation (API-003) doit désormais intégrer les priorités : position = rang réel dans l'ordre de sélection.
- Suite `sla-engine` étendue : matrice de sélection (priorités × langues × débordement) en tests table-driven, 100 % déterministes.
