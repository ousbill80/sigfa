# API-003 : Cycle de vie du ticket — émission idempotente, appel, service, clôture, no-show, transfert, abandon

**Module** : F3 — API · **Agent** : agent-api · **Dépend de** : API-002 · **Statut** : TODO (★ référence PRD)

**Révision** : v2 — arbitrage 19

## Exigences (EARS)
- QUAND un client émet un ticket (`POST /tickets`, canal+priorité LA LOI), le système doit : allouer le numéro par **lock-then-increment** (`UPDATE queues … RETURNING`, reset quotidien Abidjan au premier ticket du jour), composer `display_number` `{code}-{NNN}`, générer `tracking_id` nanoid(21), persister (phone chiffré via phone-cipher SI fourni + smsConsent), répondre 201 `{number, displayNumber, position, estimatedWaitMinutes, trackingId}` et émettre `ticket:created` + `queue:updated` en <500 ms.
- QUAND `X-Idempotency-Key` déjà vue (<24 h, Redis), retourner LA réponse originale (rejeu) ; même clé + payload différent → 409 `IDEMPOTENCY_CONFLICT` ; clé absente sur mutation critique → 400 `IDEMPOTENCY_KEY_REQUIRED`.
- `POST /counters/:id/call-next` : sélection FIFO simple dans les services du guichet (les priorités fines = API-004 qui remplace la stratégie par injection) ; SI aucun éligible → 404 `QUEUE_EMPTY` ; sinon WAITING→CALLED + `ticket:called` <500 ms. `POST /tickets/:id/call` : appel ciblé, 409 `TICKET_ALREADY_CLAIMED` si pris (verrou Redis SET NX — le lock complet 2-agents est durci en API-006).
- Machine à états STRICTE (LA LOI) : serve (CALLED→SERVING), close† (SERVING→DONE, calcule wait/service_time, `ticket:closed`), no-show (CALLED→NO_SHOW après `no_show_timeout_minutes` de la banque — 422 avant), transfer (→TRANSFERRED + réinsertion WAITING file cible + ligne `ticket_transfers`), abandon (WAITING/CALLED→ABANDONED). TOUTE transition illégale → 409 `ILLEGAL_TRANSITION`. `GET /tickets/:id` : position temps réel.
- La suite `sla-engine` doit couvrir la machine à états EXHAUSTIVEMENT (7 états × transitions, légales et illégales) + calculs wait/service exacts (fake timers).
- position = **PULL** : rang calculé par `rank() OVER (PARTITION BY queue_id ORDER BY priority DESC, issued_at)` parmi les WAITING de la file (priorité intégrée en API-004) — implémenté dans `services/queue-strategy.ts` ; interface `type TicketSelector = (queueId, counterId, tx) => Promise<Ticket|null>` ; API-003 fournit `selectNextFifo`. `queue:updated` ne porte que `length` + `estimate` (pas de liste de tickets).
- estimatedWaitMinutes = position × **TMT glissant** du service : moyenne simple des `service_time` des tickets DONE du service sur les **60 dernières minutes (fenêtre Abidjan, ≥5 observations)** — sinon `sla_minutes` du service (NOT NULL en base) — sinon défaut global 15 min. Calcul dans `services/queue-estimation.ts`. **Cache Redis TTL 10 s invalidé sur toute mutation de file.**
