## Exigences (EARS)
- **Wiring** : `const server = serve(honoApp)` (@hono/node-server) puis `new Server(server)` — HTTP et upgrade WebSocket coexistent sur le **même port** ; le handshake JWT est authentifié **AVANT** le join de room.
- Serveur Socket.io 4 attaché à l'API : handshake authentifié JWT (utilisateur OU session borne) ; un socket ne peut joindre QUE `agency:{id}` de son scope (tentative hors scope → refus + log).
- CHAQUE événement émis passe par un émetteur typé qui valide le payload contre `payloadSchema` Zod du contrat (realtime.ts) — payload invalide : événement NON émis + erreur loggée (jamais de payload sauvage).
- Adapter **Redis pub/sub** (multi-instance ready) ; émissions branchées sur API-003/004/005 (created/called/closed/queue:updated/counter:status).
- **Verrou d'appel durci** : QUAND 2 agents font call-next/call simultanément, UN SEUL obtient le ticket (Redis SET NX PX 5000 par ticket + re-vérification transactionnelle du statut) ; le perdant de call-next reçoit LE SUIVANT (pas une erreur) ; le perdant de call ciblé reçoit 409 `TICKET_ALREADY_CLAIMED`. Test de course RÉEL (≥20 paires concurrentes, zéro double-attribution).
- **Resync** : `sync:request` → `sync:state` (état complet de la file de l'agence : tickets WAITING/CALLED, compteurs, guichets) — contrat 002 ; reconnexion testée.
- Suite `realtime-guarantees` branchée : `ticket:called` reçu par un client local en <500 ms (TICKET_CALLED_SLA_MS) mesuré sur 50 émissions, p95.
